# Product

## Register

product

## Users

Loan officers and bank staff at a financial institution. Use context: office workstation, multiple tabs open, task-focused workflow. Primary job: register new mortgage clients, review client profiles, and run loan simulations. Speed and data clarity matter more than visual richness.

## Product Purpose

Internal tool for managing mortgage credit clients. Allows staff to create client records, list and search clients, and simulate French-amortization loan scenarios. Success: a loan officer can register a new client and run a simulation in under two minutes without errors.

## Brand Personality

Precise, efficient, trusted. Three words: sharp, credible, calm. No warmth-for-warmth's-sake; the credibility of a tool that does not get in the way.

## Anti-references

- Nequi, Nubank, Rappi: consumer fintech with gradient-heavy, app-store aesthetics. Wrong register.
- SAP, Oracle ERP: enterprise bloat, visual clutter, dated chrome. Opposite failure mode.
- Generic React admin dashboards with cream backgrounds and card-grid everything.

## Design Principles

1. **Hierarchy through space, not decoration.** Breathing room is the design. No borders where whitespace suffices.
2. **Motion earns its place.** Every animation must answer "what does this communicate?" Entrance, feedback, state transition only. No ambient loops.
3. **Precision over charm.** The tool should feel like a well-calibrated instrument: consistent radii, locked spacing scale, a single accent color used identically everywhere.
4. **Content is primary.** Typography and spacing serve the data in the table, not the brand. The client's name should be the most readable thing on the list screen.
5. **Fail clearly, succeed quietly.** Errors are immediate and specific. Success is a brief confirmation, then out of the way.

## Accessibility & Inclusion

WCAG AA minimum. Focus rings always visible and high-contrast. Reduced-motion preference respected for all transitions. Color is never the sole differentiator for state (error = border + icon + text, not border color alone).
